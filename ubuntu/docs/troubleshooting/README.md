## 📋 故障排查流程

### 标准排查步骤

1. **问题识别**
   - 明确问题现象
   - 记录错误信息
   - 确定影响范围

2. **信息收集**
   - 查看系统日志
   - 检查服务状态
   - 收集配置���息

3. **原因分析**
   - 查找可能原因
   - 参考历史案例
   - 分析系统变更

4. **解决方案**
   - 制定解决步骤
   - 备份重要数据
   - 执行修复操作

5. **验证测试**
   - 验证问题解决
   - 检查副作用
   - 更新配置文档

### 快速诊断命令

```bash
# 系统状态概览
systemctl --failed
journalctl -p err --since "1 hour ago"

# 网络连接检查
ping -c 4 8.8.8.8
nslookup google.com

# 磁盘使用检查
df -h
du -sh /var/log/

# 内存使用检查
free -h
top -o %MEM

# 进程状态检查
ps aux | grep defunct
systemctl list-units --failed
```
