**一致性** > **可读性** > **完整性** > **可维护性**

### 标准文档模板
```markdown

- **最后更新**: 2025-11-05

## 目录
1. [概述](#概述)

## 🔧 常用语法

### 标题
```markdown
# H1 (文档标题)
## H2 (主要章节)
### H3 (子章节)
#### H4 (详细说明)
##### H5 (小节)
###### H6 (最小标题)
```

### 文本格式
```markdown
**粗体** - 重要内容
*斜体* - 强调内容
***粗斜体*** - 特殊强调
~~删除线~~ - 废弃内容
`代码` - 命令或变量
```

### 列表
```markdown
- 无序列表
  - 子项目
  - 子项目

1. 有序列表
   1. 子步骤
   2. 子步骤

- [ ] 任务清单
- [x] 已完成任务
- [ ] 待完成任务
```

### 代码块
```bash
# Bash/Shell
command --option
```

```python
# Python代码
def function():
    print("Hello")
```

```json
{
  "key": "value"
}
```

### 表格
```markdown
| 功能 | 状态 | 描述 | 优先级 |
|------|------|------|--------|
| Docker | ✅ 完成 | 容器化部署 | 高 |
| Jenkins | 📝 进行中 | CI/CD配置 | 中 |
```

### 链接
```markdown
[内部链接](../docs/文档.md)
[章节链接](#章节标题)
[外部链接](https://example.com)
[图片链接](./images/图片.png)
```

### 引用
```markdown
> 普通引用

> > 嵌套引用
```

---

### 分隔线
```markdown

主要内容分隔
```

## 🎨 格式检查清单

### ✅ 必须检查项目
- [ ] 文档编号符合规范（三位数字）
- [ ] 标题层级正确（不超过6级）
- [ ] 文件名使用连字符分隔
- [ ] 代码块有语言标识符
- [ ] 外部链接使用https
- [ ] 相对链接使用正确路径
- [ ] 表格格式规范

### 🔍 可选检查项目
- [ ] 元数据信息完整
- [ ] 目录自动生成或手动维护
- [ ] 图片包含alt文本
- [ ] 引用来源正确标注
- [ ] 版本历史记录完整

## 🚀 常见问题

### Q: 如何处理长文档？
A: 使用清晰的章节划分，每个主要功能独立成节

### Q: 如何处理代码示例？
A: 确保代码完整可执行，包含必要注释

### Q: 如何处理链接？
A: 优先使用相对链接，外部链接确保有效性

### Q: 如何处理表格？
A: 保持表格简洁，复杂表格考虑拆分

### Q: 如何处理图片？
A: 使用相对路径，提供alt文本，控制图片大小
